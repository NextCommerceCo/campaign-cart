---
title: Cart Toggle
description: A button that adds a package when it's not in the cart, and removes it when it is. Ideal for order bumps and optional add-ons.
---

`CartToggleEnhancer` creates a toggle button. Clicking it adds the package if it isn't in the cart, and removes it if it is. The button text updates automatically to reflect the current cart state.

---

## Basic usage

```html
<button data-next-toggle
        data-next-package-id="99"
        data-add-text="Add Protection Plan (+$9)"
        data-remove-text="Protection Plan Added ✓">
  Add Protection Plan (+$9)
</button>
```

---

## Auto-add on page load

Use `data-next-selected="true"` to add the package to the cart automatically when the page loads:

```html
<button data-next-toggle
        data-next-package-id="99"
        data-next-selected="true"
        data-add-text="Add Protection Plan"
        data-remove-text="Protection Plan Added ✓">
  Protection Plan Added ✓
</button>
```

---

## Attributes

| Attribute | Description |
|---|---|
| `data-next-toggle` | Required. Marks this as a toggle button |
| `data-next-package-id` | Package to toggle. Can also be on a parent container |
| `data-next-quantity` | Quantity when adding (default `1`) |
| `data-next-selected="true"` | Automatically add to cart on page load |
| `data-add-text` | Button text when package is **not** in cart |
| `data-remove-text` | Button text when package **is** in cart |
| `data-next-is-upsell="true"` | Mark as upsell item when adding |

---

## Package ID from container

The package ID can be on the button or on any ancestor container:

```html
<div data-next-package-id="99">
  <p>Extended warranty — +$9</p>
  <button data-next-toggle
          data-add-text="Add to order"
          data-remove-text="Added ✓">
    Add to order
  </button>
</div>
```

---

## Quantity sync (warranty / bundle pattern)

Use `data-next-package-sync` to mirror the total quantity of other packages. When those packages are removed, this item is removed too. This is useful for warranties or accessories that should match the quantity of the main product.

```html
<!-- One warranty per unit of packages 10 and 20 -->
<button data-next-toggle
        data-next-package-id="99"
        data-next-package-sync="10,20"
        data-add-text="Add warranty"
        data-remove-text="Warranty added ✓">
  Add warranty
</button>
```

When the combined quantity of packages 10 and 20 changes, the warranty quantity automatically matches. If packages 10 and 20 are both removed from the cart, the warranty is removed too.

---

## CSS classes

Managed on both the button element and the nearest state container (`data-next-toggle-container`, `data-next-bump`, `data-next-upsell-item`, or ancestor with `data-next-package-id`):

| Class | Meaning |
|---|---|
| `next-in-cart` | Package is currently in the cart |
| `next-not-in-cart` | Package is not in the cart |
| `next-active` | Alias for `next-in-cart` |

```css
/* Highlight the bump when it's selected */
[data-next-bump].next-in-cart {
  border: 2px solid green;
}
```

## Order bump example

```html
<div data-next-bump data-next-package-id="99">
  <label>
    <input type="checkbox" disabled>
    <strong>Add 1-Year Warranty — only $9</strong>
    <p>Covers all defects and accidental damage.</p>
  </label>
  <button data-next-toggle
          data-add-text="Yes, add warranty!"
          data-remove-text="Warranty added ✓">
    Yes, add warranty!
  </button>
</div>
```
