## Installation

Add the SDK script to every page that uses cart or checkout functionality:

```html
<script src="https://cdn.29next.com/campaign-cart/latest/sdk.js"></script>
```

The SDK auto-initializes on `DOMContentLoaded`. No extra JavaScript needed.

---

## Configuration

The SDK reads configuration from `<meta>` tags in your `<head>`:

```html
<head>
  <!-- Required: your campaign's API key -->
  <meta name="next-api-key" content="YOUR_API_KEY">

  <!-- Required: your campaign slug -->
  <meta name="next-campaign" content="your-campaign-slug">

  <!-- Optional: currency (auto-detected from browser if omitted) -->
  <meta name="next-currency" content="USD">

  <!-- Optional: language -->
  <meta name="next-language" content="en">
</head>
```

<Callout type="warn">
  The `next-api-key` and `next-campaign` meta tags are required. The SDK will not initialize without them.
</Callout>

## Quick-start example

A minimal product page with a package selector, price display, and buy button:

```html
<!DOCTYPE html>
<html>
<head>
  <meta name="next-api-key" content="YOUR_API_KEY">
  <meta name="next-campaign" content="your-campaign">
</head>
<body>

  <!-- 1. Package selector — selecting a card swaps the cart item -->
  <div data-next-cart-selector
       data-next-selector-id="main"
       data-next-selection-mode="select">

    <div data-next-selector-card
         data-next-package-id="10"
         data-next-selected="true">
      <span>1 Bottle — $29</span>
    </div>

    <div data-next-selector-card
         data-next-package-id="20">
      <span>3 Bottles — $69</span>
    </div>

  </div>

  <!-- 2. Display the selected package price, updates on selection change -->
  <p>Total: <span data-next-display="selection.main.total"></span></p>

  <!-- 3. Buy button — reads the selection and redirects to /checkout -->
  <button data-next-action="add-to-cart"
          data-next-selector-id="main"
          data-next-url="/checkout">
    Buy Now
  </button>

  <script src="https://cdn.29next.com/campaign-cart/latest/sdk.js"></script>
</body>
</html>
```

When the user clicks **Buy Now**:
1. The selected package is added to the cart
2. The browser navigates to `/checkout` (with the cart state preserved)

## Page types

The SDK supports three main page types:

| Page | Purpose | Key enhancers |
|---|---|---|
| **Sales / Landing page** | Product presentation and package selection | `PackageSelectorEnhancer`, `AddToCartEnhancer`, display enhancers |
| **Checkout page** | Collect shipping/payment and submit order | `CheckoutFormEnhancer` |
| **Post-purchase page** | Thank you page + upsell offers | `OrderDisplayEnhancer`, `AcceptUpsellEnhancer` |

Cart state persists across pages using `sessionStorage`, so a user who selects a package on the sales page will still have it in their cart on the checkout page.

---

## URL parameters

The SDK reads several URL parameters automatically:

| Parameter | Effect |
|---|---|
| `?currency=EUR` | Override the campaign currency |
| `?ref_id=ORDER123` | Load order data on post-purchase pages |
| `?debug=true` | Enable the debug panel overlay |
| `?profile=wholesale` | Activate a pricing profile |

---

## What happens on init

When the page loads, the SDK:

1. Reads `<meta>` configuration
2. Detects visitor location and currency
3. Loads campaign data (packages, pricing, shipping) from the API (10-min session cache)
4. Rehydrates the cart from `sessionStorage`
5. Scans the DOM for `data-next-*` attributes and initializes each enhancer
6. Fires `window.nextReady` callbacks

To run code after the SDK is ready:

```js
window.nextReady = window.nextReady || [];
window.nextReady.push(function(sdk) {
  console.log('SDK ready, campaign:', sdk.getCampaignData());
});
```
